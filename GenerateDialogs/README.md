## Описание
Это папка отвечает за генерацию датасета медицинских диалогов.

DataPI.csv - база данных. В ней 16 полей-классов персональных данных. Всего 10000 строк.

scenarios.txt - Набор сценариев. Нужен для генерации диалогов ЛЛМкой.

scr_generate_llm_requests.py - основной скрипт для генерации диалогов. Он обращается к yandex yandexgpt-5.1. В этой же программе нужно указать количество генерируемых даилогов, API ключ и folder-id. Работает так: случайно решаем, использовать ли сценарий, если да, то выбираем слуйчайный из имеющихся. Если нет, то идем к другой LLM за сценарием. Дальше LLM генерит какой - то разговор, размеченный метками [S], посел чего алгоритмом проходимся и отмечаем индексы span'ов.

extract_subdialogs.py - скрипт для выделения поддиалогов. LLM генерит большие диалоги, некоторые из которых похожи друг на друга. Выделение поддиалогов помогает разнообразить диалоги, сделать их меньше (по 1 - 2 span'а).

filter_duplicates.py - скрипт для фильтрации полученных поддиалогов. Есть проблема с генерацией диалогов у yandexgpt5.1, она может сгенерировать в одном предложении два ФИО подряд или какие - то странные конструкции, которые не могут встретиться в реальной речи. Подобные случаи убираются этим скриптом. Примерно 4%-10% бракуется.

cleaning_punctuation.py - скрипт для финального причесывания. Он убирает пунктуацию и прочее.

remove_newlines.py - скрипт для приведения к сплошному потоку речи, удаляет все переводы строк.

validate_spans.py - скрипт для проверки разметки индексов span'ов. Папка для него - validation_errors.


pipeline.py - пайплаин для генерации.


## Запуск
Сначала нужно выставить требуемые данные в scr_generate_llm_requests.py.

Запускать командой python3 pipeline.py.

Итоговые файлы - final_cleaned_subdialogs.jsonl, final_cleaned_subdialogs.txt. Логи pipeline.py выводятся в консоли.
