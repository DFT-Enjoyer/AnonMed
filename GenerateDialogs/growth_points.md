## Проблемы и способы их решения
У генерации есть несколько проблем и их решений:

0) Для генерации нужны токены. У нас был ограниченный бюджет, грант не получилось использовать.

1) Генерация диалогов ЛЛМ. Промт слишком большой, модель может написать откровенно мусорные диалоги. Также сама модель довольно плохо справляется. При наличии более мощной ЛЛМ, думаю, диалоги станут более похожими на реальность. Чтобы компенсировать проблемы текущей ЛЛМ используется фильтрация, она удаляет некоторые странные патерны, которые были выявлены ручным просмотром получившихся диалогов. Также можно генерировать запросы llm следующим образом: использовать две llm. Одна генерит диалог, а этот же диалог оценивает другая модель. Если не выполняются требования, то второая модель не берет его. Или можно одной LLM генерировать сценарий, а дальше его передавать другой модели.

2) Ограниченность сырого датасета. Все цифровые данные генерируются через специальные генераторы. На данный момент они не связаны между собой, а также с буквенном форматоым. Сам сырой датасет делается так: выбирается n случайных данных из столбцов (найденные в открытом доступе датасете) из формируется один большой датасет DataPI.csv на n строк. В текущей реализации в папке не лежит способ генерации этого датасета, но его можно найти в других папках.

3) Нет метрики качества генерируемых диалогов. Для этого можно смотреть на ститистику персональных данных (сколько они занимают букв в тексте) или делать прочие оценки.
