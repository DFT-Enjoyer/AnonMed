# Оценка качества синтетических медицинских диалогов

Скрипт `metrics_dialogs.py` вычисляет набор метрик для корпуса диалогов, хранящихся в формате JSONL.  
Метрики ориентированы на анализ коротких сценариев сбора персональных данных (ПДн) и позволяют оценить как лексическое разнообразие, так и структурную вариативность формулировок.

Все метрики рассчитываются без использования эталонных диалогов — анализируется только предоставленный корпус.

## Используемые метрики

### Distinct-1 и Distinct-2
**Что измеряют:** долю уникальных слов (Distinct-1) и уникальных двухсловных сочетаний (Distinct-2) среди всех слов/биграмм в репликах.  
**Интерпретация:**
- Высокие значения свидетельствуют о богатстве лексики и слабой предсказуемости текста.
- Для коротких медицинских диалогов на русском языке ориентировочные границы: Distinct-1 > 0.25 считается приемлемым, > 0.30 — хорошим; Distinct-2 > 0.55 — приемлемо, > 0.65 — хорошо.
- Низкие значения (Distinct-1 < 0.20, Distinct-2 < 0.50) говорят о сильной повторяемости одних и тех же слов/фраз, что типично для жёстко шаблонных генераций.

### Междометная насыщенность (interjection_ratio)
**Что измеряет:** долю реплик, содержащих хотя бы один разговорный маркер (ага, угу, ну, э-э, ммм, так, ой и т.п.).  
**Интерпретация:**
- Прямой индикатор естественности устной речи. В живом диалоге спонтанные реакции, хезитации и короткие подтверждения составляют заметную часть реплик.
- Значения ниже 0.20 обычно соответствуют формальному, механическому обмену репликами.  
- Диапазон 0.25–0.40 характерен для достаточно живых диалогов без избыточной эмоциональности. Значения >0.40 могут указывать на некоторую нарочитость, но в небольших корпусах допустимы.

### Средняя длина реплики (avg_replica_length)
**Что измеряет:** среднее количество слов в одной реплике.  
**Интерпретация:**
- Короткие реплики (5–8 слов) способствуют динамичности и приближают диалог к устному обмену.  
- Слишком короткие реплики (3–4 слова) могут делать диалог рубленым, а очень длинные (>12) — превращать его в монолог.
- Для сценариев регистрации и сверки ПДн типичная длина составляет 6–9 слов.

### Mean Segmental Type‑Token Ratio (MSTTR)
**Что измеряет:** усреднённое отношение числа уникальных слов к числу всех слов в последовательных сегментах фиксированной длины (по умолчанию 20 слов).  
**Интерпретация:**
- В отличие от глобального TTR, практически не зависит от общего объёма текста и лучше отражает локальное лексическое богатство.
- Высокий MSTTR (>0.80) в коротких диалогах о ПДн ожидаем, поскольку каждая реплика часто содержит уникальные идентификаторы (фамилии, адреса, номера документов).  
- Значения ниже 0.65 наводят на мысль о частом повторении одних и тех же фраз даже внутри небольших окон.

### Self-BLEU
**Что измеряет:** среднюю степень n‑граммного сходства одного диалога со всеми остальными диалогами корпуса (на уровне слов).  
**Интерпретация:**
- Низкий Self-BLEU (<0.30) означает, что диалоги слабо похожи друг на друга, структура и лексика варьируются.  
- Значения 0.30–0.50 — умеренная схожесть, часто встречается в корпусах с преобладанием однотипных коротких сценариев.  
- Self-BLEU >0.50 сигнализирует о выраженной шаблонности; многие диалоги построены по одной канве.
- Следует учитывать, что на очень коротких диалогах Self-BLEU может быть несколько завышен из‑за совпадения приветствий и служебных слов; интерпретировать его лучше совместно с другими метриками.

### Слотовое разнообразие (slot_avg_unique и slot_coverage)
**Что измеряет:**
- `slot_coverage` — доля типов персональных данных (из заданного набора из 16 ключей), которые реально встречаются в корпусе.
- `slot_avg_unique` — среднее число уникальных формулировок, используемых для запроса/указания одного типа ПДн.
- Для каждого ключа также выводится `slot_<key>_uniq` — количество уникальных контекстов для этого конкретного поля.

**Интерпретация:**
- `slot_coverage = 1.0` означает, что все ожидаемые типы ПДн задействованы; пропуск некоторых ключей (например, `driver_license` или `mse`) может быть оправдан спецификой набора сценариев.
- `slot_avg_unique` отражает вариативность формулировок: чем выше значение, тем менее шаблонно модель запрашивает одни и те же данные.  
  Для разговорных систем значение 3–5 уникальных вариантов на слот считается хорошим, 5–10 — отличным, но практический максимум ограничен естественной синонимией (например, для «ИНН» трудно придумать больше 3–5 осмысленных вопросов).
- Сравнивайте `slot_avg_unique` с количеством диалогов: если диалогов много, а уникальных формулировок мало, значит, запросы повторяются почти без изменений.

## Требования и установка

- Python 3.7+
- Необходимые библиотеки:
  - `nltk`
  - `scikit-learn`

Установите зависимости и загрузите ресурсы NLTK:

```bash
pip install nltk scikit-learn
python -c "import nltk; nltk.download('punkt')"

python metrics_dialogs.py путь/к/диалогам.jsonl
